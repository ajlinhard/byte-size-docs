What is AWS Bedrock?

Amazon Bedrock is a fully managed service that gives you access to a wide selection of foundation models (FMs) from Amazon and leading AI providers through a single API. You don't need to manage infrastructure, train models, or handle scaling — Bedrock handles it all.

It integrates deeply with the AWS ecosystem (IAM, S3, VPC, CloudWatch), making it the natural choice for teams already on AWS who want to add generative AI to their applications.

🧠
Foundation Models
Access Claude, Titan, Llama, Mistral, Cohere, Stability AI and more via one unified API.
🗃️
Knowledge Bases
Managed RAG pipeline — connect S3 data sources, auto-sync, and query with semantic search.
🤖
Agents
Build autonomous multi-step agents that reason, plan, call tools, and take actions.
🛡️
Guardrails
Content filtering, PII redaction, topic denial, and grounding checks for safe deployments.
🎨
Image Generation
Generate, edit, and transform images with Stability AI, Amazon Nova Canvas, and more.
📊
Model Evaluation
Benchmark models on your datasets for accuracy, robustness, and safety metrics.

Setup & Authentication

Core

Before making any API calls, you need an AWS account with Bedrock access enabled. Model access must be explicitly requested in the Bedrock console for each model you want to use.

  1. 1

    Enable Bedrock Model Access

    Go to AWS Console → Amazon Bedrock → Model Access → Request access for each model.

  2. 2

    Install the AWS SDK

    Install boto3 and configure your AWS credentials (via CLI, env vars, or IAM role).

  3. 3

    Create a Bedrock client

    Use bedrock-runtime for model inference and bedrock for management operations.

setup.py 
# Install dependencies
# pip install boto3

import boto3
import json

# --- Option 1: Use default credentials (AWS CLI configured) ---
# Run: aws configure
# Credentials stored in ~/.aws/credentials

# --- Option 2: Explicit credentials (dev/testing only) ---
bedrock_runtime = boto3.client(
    service_name="bedrock-runtime",
    region_name="us-east-1",
    aws_access_key_id="YOUR_ACCESS_KEY",       # or use env var
    aws_secret_access_key="YOUR_SECRET_KEY",   # or use env var
)

# --- Option 3: IAM Role (recommended for production) ---
# Attach an IAM role with AmazonBedrockFullAccess
# boto3 auto-detects the role when running on EC2/Lambda/ECS

# Two clients you'll use:
bedrock = boto3.client("bedrock", region_name="us-east-1")              # management
bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")  # inference

print("Clients created successfully!")
IAM Permissions: Your role/user needs bedrock:InvokeModel for inference and bedrock:ListFoundationModels to discover models. Use the managed policy AmazonBedrockFullAccess for full access.

InvokeModel API

Beginner

The most direct way to call a model. You send a raw JSON body (model-specific format) and get a response back. Each model provider has its own request/response schema.

Claude (Anthropic Messages Format)

invoke_claude.py 
import boto3
import json

bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

MODEL_ID = "anthropic.claude-3-5-sonnet-20241022-v2:0"

# Build the request body (Anthropic Messages API format)
body = {
    "anthropic_version": "bedrock-2023-05-31",
    "max_tokens": 1024,
    "messages": [
        {
            "role": "user",
            "content": "Explain the difference between supervised and unsupervised learning."
        }
    ],
    "system": "You are a helpful AI tutor. Keep responses concise."
}

response = bedrock_runtime.invoke_model(
    modelId=MODEL_ID,
    body=json.dumps(body),
    contentType="application/json",
    accept="application/json"
)

# Parse the response
result = json.loads(response["body"].read())
print(result["content"][0]["text"])
print(f"\n--- Usage ---")
print(f"Input tokens:  {result['usage']['input_tokens']}")
print(f"Output tokens: {result['usage']['output_tokens']}")

Converse API

Beginner

The Converse API provides a unified, provider-agnostic interface for chat. It handles the message format differences between models, so you can swap models without changing your code.

converse_basic.py 
import boto3

bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

# --- Single turn ---
response = bedrock_runtime.converse(
    modelId="anthropic.claude-3-5-sonnet-20241022-v2:0",
    messages=[
        {"role": "user", "content": [{"text": "What is AWS Bedrock?"}]}
    ],
    system=[{"text": "You are a concise AWS expert."}],
    inferenceConfig={
        "maxTokens": 512,
        "temperature": 0.5,
        "topP": 0.9
    }
)

print(response["output"]["message"]["content"][0]["text"])

# --- Multi-turn conversation ---
conversation_history = []

def chat(user_message: str) -> str:
    conversation_history.append({
        "role": "user",
        "content": [{"text": user_message}]
    })

    response = bedrock_runtime.converse(
        modelId="anthropic.claude-3-5-sonnet-20241022-v2:0",
        messages=conversation_history,
        system=[{"text": "You are a helpful coding assistant."}],
        inferenceConfig={"maxTokens": 1024}
    )

    assistant_msg = response["output"]["message"]
    conversation_history.append(assistant_msg)

    return assistant_msg["content"][0]["text"]

# Have a multi-turn conversation
print(chat("Write a Python function to reverse a string."))
print(chat("Now add error handling for non-string inputs."))
print(chat("Add type hints and docstring.")

Streaming Responses

Beginner

Streaming lets you receive tokens as they're generated rather than waiting for the full response. This dramatically improves perceived latency for end users.

streaming.py 
import boto3
import json
import sys

bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

# --- Stream with converse_stream ---
response = bedrock_runtime.converse_stream(
    modelId="anthropic.claude-3-5-sonnet-20241022-v2:0",
    messages=[
        {"role": "user", "content": [{"text": "Write a haiku about AWS cloud computing."}]}
    ]
)

# Process the event stream
full_text = ""
for event in response["stream"]:
    if "contentBlockDelta" in event:
        delta = event["contentBlockDelta"]["delta"]
        if "text" in delta:
            token = delta["text"]
            full_text += token
            print(token, end="", flush=True)  # Print without newline for streaming effect

    elif "messageStop" in event:
        stop_reason = event["messageStop"]["stopReason"]
        print(f"\n\n[Stop reason: {stop_reason}]")

    elif "metadata" in event:
        usage = event["metadata"].get("usage", {})
        print(f"[Tokens - Input: {usage.get('inputTokens', 0)}, Output: {usage.get('outputTokens', 0)}]")


# --- Stream with invoke_model_with_response_stream ---
body = json.dumps({
    "anthropic_version": "bedrock-2023-05-31",
    "max_tokens": 1024,
    "messages": [{"role": "user", "content": "Tell me a short story about a robot."}]
})

response = bedrock_runtime.invoke_model_with_response_stream(
    modelId="anthropic.claude-3-5-sonnet-20241022-v2:0",
    body=body
)

for event in response["body"]:
    chunk = json.loads(event["chunk"]["bytes"])
    if chunk["type"] == "content_block_delta":
        print(chunk["delta"]["text"], end="", flush=True)

Embeddings

Intermediate

Embeddings convert text into dense numeric vectors that capture semantic meaning. Use them for semantic search, clustering, similarity comparison, and as input to RAG pipelines.

embeddings.py 
import boto3
import json
import numpy as np

bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

EMBED_MODEL = "amazon.titan-embed-text-v2:0"

def get_embedding(text: str, dimensions: int = 1024) -> list[float]:
    """Get a text embedding from Amazon Titan."""
    response = bedrock_runtime.invoke_model(
        modelId=EMBED_MODEL,
        body=json.dumps({
            "inputText": text,
            "dimensions": dimensions,      # 256, 512, or 1024
            "normalize": True              # L2-normalize the vector
        })
    )
    result = json.loads(response["body"].read())
    return result["embedding"]


def cosine_similarity(a: list[float], b: list[float]) -> float:
    """Compute cosine similarity between two vectors."""
    a, b = np.array(a), np.array(b)
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))


# --- Semantic similarity example ---
texts = [
    "How do I reset my password?",
    "I forgot my login credentials",
    "What is the refund policy?",
    "Can I get my money back?",
    "How to change account password"
]

query = "I can't log in to my account"
query_embedding = get_embedding(query)

results = []
for text in texts:
    embedding = get_embedding(text)
    score = cosine_similarity(query_embedding, embedding)
    results.append((score, text))

results.sort(reverse=True)
print(f"Query: '{query}'\n")
for score, text in results:
    print(f"  {score:.3f}  {text}")

# Output:
# Query: 'I can't log in to my account'
#   0.921  How do I reset my password?
#   0.907  I forgot my login credentials
#   0.898  How to change account password
#   0.612  What is the refund policy?
#   0.601  Can I get my money back?
Cohere Embed: For multilingual embeddings, use cohere.embed-multilingual-v3 which supports 100+ languages and is excellent for global applications.

Knowledge Bases (RAG)

Intermediate

Bedrock Knowledge Bases is a fully managed Retrieval-Augmented Generation (RAG) pipeline. You connect S3 data sources, Bedrock auto-chunks and embeds the content, and you query it with natural language. No vector database to manage.

📥

Data Sources

S3, Confluence, Salesforce, SharePoint, Web Crawler — all supported as ingestion sources.

🗄️

Vector Stores

OpenSearch Serverless, Aurora PostgreSQL (pgvector), Pinecone, Mongo Atlas, Redis Enterprise.

knowledge_base_rag.py 
import boto3
import json

bedrock_agent_runtime = boto3.client("bedrock-agent-runtime", region_name="us-east-1")

KNOWLEDGE_BASE_ID = "XXXXXXXXXX"   # From Bedrock console
MODEL_ID = "anthropic.claude-3-5-sonnet-20241022-v2:0"

# --- Option 1: retrieve_and_generate (fully managed RAG) ---
response = bedrock_agent_runtime.retrieve_and_generate(
    input={"text": "What is the company's remote work policy?"},
    retrieveAndGenerateConfiguration={
        "type": "KNOWLEDGE_BASE",
        "knowledgeBaseConfiguration": {
            "knowledgeBaseId": KNOWLEDGE_BASE_ID,
            "modelArn": f"arn:aws:bedrock:us-east-1::foundation-model/{MODEL_ID}",
            "retrievalConfiguration": {
                "vectorSearchConfiguration": {
                    "numberOfResults": 5,   # How many chunks to retrieve
                    "overrideSearchType": "SEMANTIC"  # or HYBRID
                }
            },
            "generationConfiguration": {
                "promptTemplate": {
                    "textPromptTemplate": """
You are a helpful HR assistant. Answer based only on the provided context.
If you cannot find the answer, say "I don't have that information."

$search_results$

Question: $query$
"""
                }
            }
        }
    }
)

print("Answer:", response["output"]["text"])
print("\nSources:")
for citation in response.get("citations", []):
    for ref in citation.get("retrievedReferences", []):
        loc = ref["location"].get("s3Location", {})
        print(f"  - {loc.get('uri', 'unknown')}")


# --- Option 2: retrieve only (bring your own generation) ---
retrieve_response = bedrock_agent_runtime.retrieve(
    knowledgeBaseId=KNOWLEDGE_BASE_ID,
    retrievalQuery={"text": "vacation days policy"},
    retrievalConfiguration={
        "vectorSearchConfiguration": {"numberOfResults": 3}
    }
)

# You get raw chunks — use them however you want
for result in retrieve_response["retrievalResults"]:
    score = result["score"]
    content = result["content"]["text"]
    source = result["location"].get("s3Location", {}).get("uri", "unknown")
    print(f"Score: {score:.3f} | Source: {source}")
    print(f"  {content[:200]}...")

Image Generation

Intermediate

Bedrock supports text-to-image, image-to-image, and inpainting via Stability AI SDXL and Amazon Nova Canvas models.

image_generation.py 
import boto3
import json
import base64
from pathlib import Path

bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

# --- Text-to-Image with Stability AI SDXL ---
def generate_image_sdxl(prompt: str, output_path: str = "output.png"):
    body = {
        "text_prompts": [
            {"text": prompt, "weight": 1.0},
            {"text": "blurry, low quality, distorted", "weight": -1.0}  # negative prompt
        ],
        "cfg_scale": 7,          # How closely to follow the prompt (1-35)
        "steps": 50,             # Inference steps
        "seed": 42,              # For reproducibility
        "width": 1024,
        "height": 1024,
        "style_preset": "photographic"  # photographic | anime | digital-art | etc.
    }

    response = bedrock_runtime.invoke_model(
        modelId="stability.stable-diffusion-xl-v1",
        body=json.dumps(body)
    )

    result = json.loads(response["body"].read())
    image_data = base64.b64decode(result["artifacts"][0]["base64"])

    Path(output_path).write_bytes(image_data)
    print(f"Image saved to {output_path}")
    return image_data


# --- Text-to-Image with Amazon Nova Canvas ---
def generate_image_nova(prompt: str, output_path: str = "nova_output.png"):
    body = {
        "taskType": "TEXT_IMAGE",
        "textToImageParams": {
            "text": prompt,
            "negativeText": "blurry, watermark, text"
        },
        "imageGenerationConfig": {
            "numberOfImages": 1,
            "width": 1024,
            "height": 1024,
            "quality": "premium",   # standard | premium
            "cfgScale": 8.0,
            "seed": 42
        }
    }

    response = bedrock_runtime.invoke_model(
        modelId="amazon.nova-canvas-v1:0",
        body=json.dumps(body)
    )

    result = json.loads(response["body"].read())
    image_data = base64.b64decode(result["images"][0])
    Path(output_path).write_bytes(image_data)
    return image_data


# --- Image variation / editing with Nova Canvas ---
def edit_image_nova(source_image_path: str, prompt: str, mask_prompt: str):
    with open(source_image_path, "rb") as f:
        image_b64 = base64.b64encode(f.read()).decode("utf-8")

    body = {
        "taskType": "TEXT_IMAGE",
        "inPaintingParams": {
            "text": prompt,
            "maskPrompt": mask_prompt,  # Describe what area to replace
            "image": image_b64
        },
        "imageGenerationConfig": {"numberOfImages": 1, "quality": "premium"}
    }

    response = bedrock_runtime.invoke_model(
        modelId="amazon.nova-canvas-v1:0",
        body=json.dumps(body)
    )
    return json.loads(response["body"].read())


# Usage
generate_image_sdxl(
    "A futuristic data center in the clouds, cyberpunk aesthetic, detailed, 8k"
)

Guardrails

Intermediate

Guardrails let you apply content policies, PII redaction, topic restrictions, and grounding checks to any model on Bedrock — independent of the model itself.

guardrails.py 
import boto3
import json

bedrock = boto3.client("bedrock", region_name="us-east-1")
bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

# --- Step 1: Create a Guardrail ---
guardrail = bedrock.create_guardrail(
    name="production-guardrail",
    description="Safety guardrail for customer-facing chatbot",

    # Block harmful content categories
    contentPolicyConfig={
        "filtersConfig": [
            {"type": "SEXUAL",     "inputStrength": "HIGH",   "outputStrength": "HIGH"},
            {"type": "VIOLENCE",   "inputStrength": "MEDIUM", "outputStrength": "MEDIUM"},
            {"type": "HATE",       "inputStrength": "HIGH",   "outputStrength": "HIGH"},
            {"type": "INSULTS",    "inputStrength": "MEDIUM", "outputStrength": "MEDIUM"},
            {"type": "MISCONDUCT", "inputStrength": "MEDIUM", "outputStrength": "MEDIUM"},
        ]
    },

    # Redact PII from both inputs and outputs
    sensitiveInformationPolicyConfig={
        "piiEntitiesConfig": [
            {"type": "EMAIL",         "action": "ANONYMIZE"},
            {"type": "PHONE",         "action": "ANONYMIZE"},
            {"type": "CREDIT_DEBIT_CARD_NUMBER", "action": "BLOCK"},
            {"type": "US_SOCIAL_SECURITY_NUMBER", "action": "BLOCK"},
            {"type": "NAME",          "action": "ANONYMIZE"},
        ]
    },

    # Restrict off-topic discussions
    topicPolicyConfig={
        "topicsConfig": [
            {
                "name": "competitor-products",
                "definition": "Questions about competing products or services",
                "examples": ["How does this compare to Salesforce?", "Is Azure better?"],
                "type": "DENY"
            }
        ]
    },

    blockedInputMessaging="I cannot process this request.",
    blockedOutputsMessaging="I cannot provide that information."
)

guardrail_id = guardrail["guardrailId"]
guardrail_version = "DRAFT"

# --- Step 2: Use guardrail in inference ---
response = bedrock_runtime.converse(
    modelId="anthropic.claude-3-5-sonnet-20241022-v2:0",
    messages=[
        {"role": "user", "content": [{"text": "My email is john@example.com. Help me with billing."}]}
    ],
    guardrailConfig={
        "guardrailIdentifier": guardrail_id,
        "guardrailVersion": guardrail_version,
        "trace": "enabled"   # Returns trace showing what was filtered
    }
)

output = response["output"]["message"]["content"][0]["text"]
print("Response:", output)
# PII will be redacted: "My email is {EMAIL}. Help me with billing."

# Check trace for filter actions
if "trace" in response:
    trace = response["trace"]["guardrail"]
    print("\nGuardrail trace:", json.dumps(trace, indent=2))

Agents

Advanced

Bedrock Agents enable autonomous, multi-step workflows. An agent can reason over a task, decide which tools to call, execute actions (via Lambda), query knowledge bases, and synthesize a final response — all without you orchestrating each step.

Architecture: You define Action Groups (with OpenAPI schemas or Lambda ARNs), optionally attach Knowledge Bases, and the agent uses ReAct-style reasoning to determine what to do at each step.
create_agent.py 
import boto3
import json

bedrock_agent = boto3.client("bedrock-agent", region_name="us-east-1")

AGENT_ROLE_ARN = "arn:aws:iam::123456789:role/BedrockAgentRole"

# --- Step 1: Create the agent ---
agent = bedrock_agent.create_agent(
    agentName="customer-support-agent",
    agentResourceRoleArn=AGENT_ROLE_ARN,
    foundationModel="anthropic.claude-3-5-sonnet-20241022-v2:0",
    description="Handles customer queries, checks order status, processes refunds.",
    instruction="""
You are a customer support agent for TechShop.
You have access to tools to:
- Look up order status by order ID
- Process refund requests
- Search the product knowledge base

Always be polite and professional. Verify customer identity before sharing order details.
If you cannot resolve an issue, escalate to a human agent.
""",
    idleSessionTTLInSeconds=600
)

agent_id = agent["agent"]["agentId"]

# --- Step 2: Add an Action Group (connects to Lambda) ---
action_group = bedrock_agent.create_agent_action_group(
    agentId=agent_id,
    agentVersion="DRAFT",
    actionGroupName="order-management",
    description="Actions for managing customer orders",
    actionGroupExecutor={
        "lambda": "arn:aws:lambda:us-east-1:123456789:function:order-management"
    },
    # OpenAPI schema defines available functions
    apiSchema={
        "payload": json.dumps({
            "openapi": "3.0.0",
            "info": {"title": "Order API", "version": "1.0.0"},
            "paths": {
                "/getOrderStatus": {
                    "get": {
                        "operationId": "getOrderStatus",
                        "description": "Get the status of a customer order",
                        "parameters": [
                            {"name": "orderId", "in": "query", "required": True,
                             "schema": {"type": "string"},
                             "description": "The order ID to look up"}
                        ],
                        "responses": {"200": {"description": "Order status"}}
                    }
                },
                "/processRefund": {
                    "post": {
                        "operationId": "processRefund",
                        "description": "Process a refund for an order",
                        "requestBody": {
                            "content": {"application/json": {"schema": {
                                "type": "object",
                                "properties": {
                                    "orderId": {"type": "string"},
                                    "reason": {"type": "string"}
                                }
                            }}}
                        },
                        "responses": {"200": {"description": "Refund status"}}
                    }
                }
            }
        })
    }
)

# --- Step 3: Prepare and deploy the agent ---
bedrock_agent.prepare_agent(agentId=agent_id)
invoke_agent.py 
import boto3
import uuid

bedrock_agent_runtime = boto3.client("bedrock-agent-runtime", region_name="us-east-1")

AGENT_ID = "XXXXXXXXXX"
AGENT_ALIAS_ID = "TSTALIASID"  # or your deployed alias

def invoke_agent(user_message: str, session_id: str = None) -> str:
    """Invoke a Bedrock Agent and stream the response."""
    if not session_id:
        session_id = str(uuid.uuid4())

    response = bedrock_agent_runtime.invoke_agent(
        agentId=AGENT_ID,
        agentAliasId=AGENT_ALIAS_ID,
        sessionId=session_id,           # Maintain conversation context
        inputText=user_message,
        enableTrace=True                # See agent's reasoning steps
    )

    full_response = ""
    trace_events = []

    for event in response["completion"]:
        # Final answer chunks
        if "chunk" in event:
            text = event["chunk"]["bytes"].decode("utf-8")
            full_response += text
            print(text, end="", flush=True)

        # Agent reasoning trace
        elif "trace" in event:
            trace = event["trace"]["trace"]
            if "orchestrationTrace" in trace:
                orch = trace["orchestrationTrace"]

                # Model's reasoning
                if "rationale" in orch:
                    print(f"\n[Reasoning] {orch['rationale']['text'][:100]}...")

                # Tool calls
                if "invocationInput" in orch:
                    inv = orch["invocationInput"]
                    if "actionGroupInvocationInput" in inv:
                        action = inv["actionGroupInvocationInput"]
                        print(f"\n[Tool call] {action['actionGroupName']}/{action['function']}")

                # Tool results
                if "observation" in orch:
                    obs = orch["observation"]
                    if "actionGroupInvocationOutput" in obs:
                        result = obs["actionGroupInvocationOutput"]["text"]
                        print(f"\n[Tool result] {result[:100]}...")

    return full_response


# Usage — the agent handles multi-step reasoning automatically
session = str(uuid.uuid4())
print(invoke_agent("What's the status of order ORD-12345?", session))
print(invoke_agent("Actually, I'd like to request a refund for that.", session))

Fine-Tuning (Model Customization)

Advanced

Fine-tune models on your own data to improve accuracy on domain-specific tasks. Bedrock supports fine-tuning for select models including Amazon Titan and Cohere Command.

Note: Fine-tuning Claude models via Bedrock is not available through the standard console as of 2024. For Claude fine-tuning, use Anthropic's platform directly. For open models like Llama, use AWS SageMaker.
fine_tuning.py 
import boto3
import json

bedrock = boto3.client("bedrock", region_name="us-east-1")

# --- Step 1: Prepare your training data in S3 ---
# Training data format for Titan (JSONL):
# {"prompt": "Classify: 'I love this product!'", "completion": "positive"}
# {"prompt": "Classify: 'Worst purchase ever'", "completion": "negative"}

# --- Step 2: Create a fine-tuning job ---
job = bedrock.create_model_customization_job(
    jobName="sentiment-classifier-v1",
    customModelName="my-sentiment-classifier",
    roleArn="arn:aws:iam::123456789:role/BedrockFineTuningRole",
    baseModelIdentifier="amazon.titan-text-lite-v1",
    customizationType="FINE_TUNING",

    trainingDataConfig={
        "s3Uri": "s3://my-bucket/training-data/sentiment_train.jsonl"
    },
    validationDataConfig={
        "validators": [{
            "s3Uri": "s3://my-bucket/training-data/sentiment_val.jsonl"
        }]
    },
    outputDataConfig={
        "s3Uri": "s3://my-bucket/fine-tuning-output/"
    },

    hyperParameters={
        "epochCount": "3",
        "batchSize": "8",
        "learningRate": "0.00001",
        "learningRateWarmupSteps": "100"
    }
)

job_arn = job["jobArn"]
print(f"Fine-tuning job started: {job_arn}")

# --- Step 3: Monitor job progress ---
import time

while True:
    status_response = bedrock.get_model_customization_job(jobIdentifier=job_arn)
    status = status_response["status"]
    print(f"Status: {status}")

    if status in ["Completed", "Failed", "Stopped"]:
        break
    time.sleep(60)

if status == "Completed":
    custom_model_arn = status_response["outputModelArn"]
    print(f"Custom model ARN: {custom_model_arn}")

    # --- Step 4: Deploy the fine-tuned model ---
    # You need Provisioned Throughput to use custom models
    throughput = bedrock.create_provisioned_model_throughput(
        modelUnits=1,
        provisionedModelName="my-sentiment-model",
        modelId=custom_model_arn
    )
    provisioned_arn = throughput["provisionedModelArn"]

    # --- Step 5: Use the fine-tuned model ---
    bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")
    response = bedrock_runtime.invoke_model(
        modelId=provisioned_arn,
        body=json.dumps({
            "inputText": "Classify: 'This app crashes constantly!'",
            "textGenerationConfig": {"maxTokenCount": 10}
        })
    )
    result = json.loads(response["body"].read())
    print(result["results"][0]["outputText"])  # → "negative"

Batch Inference

Advanced

Batch inference processes large datasets asynchronously at significantly lower cost (up to 50% discount). Ideal for offline processing jobs — embeddings, classifications, summarization at scale.

batch_inference.py 
import boto3
import json

bedrock = boto3.client("bedrock", region_name="us-east-1")
s3 = boto3.client("s3")

BUCKET = "my-batch-bucket"
ROLE_ARN = "arn:aws:iam::123456789:role/BedrockBatchRole"

# --- Step 1: Prepare input JSONL file ---
# Each line: {"recordId": "...", "modelInput": {...}}

def prepare_batch_input(records: list[dict], output_key: str = "batch_input.jsonl"):
    """Write records to S3 in the required JSONL format."""
    lines = []
    for i, record in enumerate(records):
        lines.append(json.dumps({
            "recordId": str(i),
            "modelInput": {
                "anthropic_version": "bedrock-2023-05-31",
                "max_tokens": 256,
                "messages": [{"role": "user", "content": record["text"]}]
            }
        }))

    s3.put_object(
        Bucket=BUCKET,
        Key=output_key,
        Body="\n".join(lines).encode("utf-8")
    )
    return f"s3://{BUCKET}/{output_key}"


# Example: Batch-classify 10,000 support tickets
records = [
    {"text": "Classify this support ticket as Bug/Feature/Question: 'App crashes on login'"},
    {"text": "Classify this support ticket as Bug/Feature/Question: 'Can you add dark mode?'"},
    # ... thousands more
]

input_s3_uri = prepare_batch_input(records)

# --- Step 2: Submit the batch job ---
batch_job = bedrock.create_model_invocation_job(
    jobName="ticket-classification-batch",
    modelId="anthropic.claude-3-haiku-20240307-v1:0",   # Haiku is cost-effective for batch
    roleArn=ROLE_ARN,
    inputDataConfig={
        "s3InputDataConfig": {
            "s3Uri": input_s3_uri,
            "s3InputFormat": "JSONL"
        }
    },
    outputDataConfig={
        "s3OutputDataConfig": {
            "s3Uri": f"s3://{BUCKET}/batch-results/",
            "s3EncryptionKeyId": None  # Optional: KMS key for encryption
        }
    },
    timeoutDurationInHours=24  # Max time to complete
)

job_arn = batch_job["jobArn"]
print(f"Batch job submitted: {job_arn}")

# --- Step 3: Monitor and collect results ---
import time

while True:
    status = bedrock.get_model_invocation_job(jobIdentifier=job_arn)
    print(f"Status: {status['status']} | "
          f"Input: {status.get('inputTokenCount', 0)} tokens")

    if status["status"] in ["Completed", "Failed", "Stopped", "PartiallyCompleted"]:
        break
    time.sleep(60)

# --- Step 4: Parse output JSONL ---
# Output file: s3://my-batch-bucket/batch-results/{job_id}/output.jsonl
# Each line: {"recordId": "0", "modelOutput": {...}}

def parse_batch_results(s3_output_uri: str):
    bucket, key = s3_output_uri.replace("s3://", "").split("/", 1)
    obj = s3.get_object(Bucket=bucket, Key=key)
    results = []
    for line in obj["Body"].read().decode("utf-8").strip().split("\n"):
        record = json.loads(line)
        record_id = record["recordId"]
        output = record["modelOutput"]["content"][0]["text"]
        results.append({"id": record_id, "classification": output.strip()})
    return results
Cost savings: Batch inference typically costs 50% less than on-demand. For 1M tokens with Claude Haiku, you'd pay ~$0.125 for batch vs ~$0.25 on-demand.
